# コードレビュー

コミット前の変更に対する包括的なセキュリティと品質レビュー：

1. 変更されたファイルを取得: git diff --name-only HEAD

2. 変更されたファイルごとに、以下の項目をチェック:

**セキュリティ問題（重大）:**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクションの脆弱性
- XSS（クロスサイトスクリプティング）の脆弱性
- 入力値検証の欠落
- セキュアでない依存関係
- ディレクトリトラバーサルのリスク

**コード品質（高）:**
- 50行以上の関数
- 800行以上のファイル
- 4段階以上のネストの深さ
- エラーハンドリングの欠落
- console.logステートメント
- TODO/FIXMEコメント
- 公開API用のJSDocの欠落

**ベストプラクティス（中）:**
- ミューテーションパターン（代わりに不変パターンを使用）
- コード/コメント内の絵文字の使用
- 新規コードのテスト欠落
- アクセシビリティの問題（a11y）

3. 以下の内容でレポートを生成:
   - 重大度: CRITICAL（重大）、HIGH（高）、MEDIUM（中）、LOW（低）
   - ファイルの場所と行番号
   - 問題の説明
   - 修正案

4. 重大度がCRITICALまたはHIGHの問題が見つかった場合はコミットをブロック

セキュリティ脆弱性があるコードは決して承認しないでください！

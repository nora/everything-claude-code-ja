# テストカバレッジ

テストカバレッジを分析して、不足しているテストを生成します：

1. カバレッジ付きでテストを実行: npm test --coverage または pnpm test --coverage

2. カバレッジレポートを分析（coverage/coverage-summary.json）

3. カバレッジが80%以下のファイルを特定

4. カバレッジが不足しているファイルごとに：
   - テストされていないコードパスを分析
   - 関数用のユニットテストを生成
   - API用の統合テストを生成
   - 重要なフロー用のE2Eテストを生成

5. 新しいテストが成功することを確認

6. 翻訳前後のカバレッジメトリクスを表示

7. プロジェクト全体のカバレッジが80%以上に到達することを確認

重点項目：
- 正常系シナリオ
- エラー処理
- エッジケース（null、undefined、空）
- 境界条件

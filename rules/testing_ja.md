# テスト要件

## 最小テスト カバレッジ: 80%

テスト種別（すべて必須）:
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **統合テスト** - APIエンドポイント、データベース操作
3. **E2Eテスト** - 重要なユーザーフロー（Playwright）

## テスト駆動開発

必須ワークフロー:
1. テストを最初に記述（RED）
2. テストを実行 - FAILすべき
3. 最小限の実装を記述（GREEN）
4. テストを実行 - PASSすべき
5. リファクタリング（IMPROVE）
6. カバレッジを確認（80%以上）

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使用
2. テストの分離を確認
3. モックが正しいことを確認
4. 実装を修正、テストではなく（テストが間違っていない限り）

## エージェント サポート

- **tdd-guide** - 新機能のため主体的に使用、テスト最初の記述を強制
- **e2e-runner** - Playwright E2Eテストの専門家

# エージェントオーケストレーション

## 利用可能なエージェント

`~/.claude/agents/` に格納されています：

| エージェント | 目的 | 使用時機 |
|-------|---------|-------------|
| planner | 実装計画 | 複雑な機能、リファクタリング |
| architect | システム設計 | アーキテクチャの決定 |
| tdd-guide | テスト駆動開発 | 新機能、バグ修正 |
| code-reviewer | コードレビュー | コード作成後 |
| security-reviewer | セキュリティ分析 | コミット前 |
| build-error-resolver | ビルドエラーの修正 | ビルド失敗時 |
| e2e-runner | E2Eテスト | 重要なユーザーフロー |
| refactor-cleaner | 不要コードの削除 | コード保守 |
| doc-updater | ドキュメンテーション | ドキュメント更新 |

## 即座のエージェント使用

ユーザープロンプト不要：
1. 複雑な機能リクエスト - **planner** エージェントを使用
2. コード作成・修正直後 - **code-reviewer** エージェントを使用
3. バグ修正または新機能 - **tdd-guide** エージェントを使用
4. アーキテクチャの決定 - **architect** エージェントを使用

## 並列タスク実行

独立した操作には、**常に** 並列タスク実行を使用してください：

```markdown
# 良い例：並列実行
3つのエージェントを並列で起動：
1. エージェント 1：auth.ts のセキュリティ分析
2. エージェント 2：キャッシュシステムのパフォーマンスレビュー
3. エージェント 3：utils.ts の型チェック

# 悪い例：不要な順序実行
まずエージェント 1、その次にエージェント 2、その次にエージェント 3
```

## マルチパースペクティブ分析

複雑な問題に対しては、複数の視点を持つサブエージェントを使用してください：
- 事実確認者
- シニアエンジニア
- セキュリティエキスパート
- 一貫性レビュアー
- 冗長性チェッカー

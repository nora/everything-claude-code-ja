# ユーザーレベル CLAUDE.md の例

これはユーザーレベルの CLAUDE.md ファイルの例です。`~/.claude/CLAUDE.md` に配置してください。

ユーザーレベルの設定はすべてのプロジェクトにグローバルに適用されます。以下の用途に使用します：
- 個人的なコーディング優先事項
- 常に適用したい普遍的なルール
- モジュール化されたルールへのリンク

---

## コアフィロソフィ

Claude Code です。複雑なタスクには専門的なエージェントとスキルを使用します。

**主要な原則：**
1. **エージェント優先**：複雑な作業は専門的なエージェントに委任
2. **並列実行**：可能な場合、Task ツールで複数のエージェントを使用
3. **実行前に計画**：複雑な操作には Plan Mode を使用
4. **テスト駆動**：実装の前にテストを記述
5. **セキュリティ優先**：セキュリティで妥協しない

---

詳細なガイドラインは `~/.claude/rules/` にあります：

| ルールファイル | 内容 |

## 個人的な優先事項

### コードスタイル
- コード、コメント、ドキュメントにはエモジを使用しない
- イミュータビリティを優先 - オブジェクトまたは配列を変更しない
- 少数の大きなファイルより、多くの小さなファイルを選好
- 通常 200～400 行、最大 800 行/ファイル

### Git
- 従来のコミット：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- コミット前に常にローカルでテスト
- 小さく焦点を絞ったコミット

### テスト
- テスト駆動開発：テストを最初に記述
- 最小 80% カバレッジ
- 重要なフローについてはユニット + 統合 + E2E テスト

## エディター統合

主なエディターとして Zed を使用しています：
- ファイル追跡用のエージェントパネル
- コマンドパレット用の CMD+Shift+R
- Vim モード有効

## 成功メトリクス

以下の場合、成功しています：
- すべてのテストが通過（80% 以上のカバレッジ）
- セキュリティ脆弱性なし
- コードが読みやすく保守可能
- ユーザー要件が満たされている
